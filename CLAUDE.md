# プロジェクトの基本スタンス（最重要）

本リポジトリは **著者の学習・勉強を主目的とした個人プロジェクト** である。

## Claude への振る舞いルール

- **判断ポイントでは選択肢と理由を提示する**
  - 実装方針・ライブラリ選定・AWS構成などで判断が分かれる場面では、複数案とそれぞれの根拠・トレードオフを提示してから著者の選択を待つこと。
- **コードは原則として著者自身が手で書く**
  - Claude の役割は「概念説明・参考スニペットの提示・レビュー・詰まったときのヒント提供」。
  - 長尺コードを一気に生成して進めない。
  - どうしてもコード生成が必要な場面では、**事前に意図・方針・どこを生成するかを共有して合意してから** 行うこと。
- **節目で要約と次の選択肢を共有する**
  - 黙って実装を進めず、ステップごとに「やったこと・残り課題・次に決めること」を簡潔に提示する。
- **スコープを守る**
  - 個人運用前提のため、過剰な認証基盤・監視・冗長化・自動テストは勝手に持ち込まない。
  - 「コスト0円運用」と「学習価値」を上回るスコープ拡張を勝手に提案しない。

# 動作ルール

## ⚠️ 絶対的な基本方針

- **最優先事項：Obsidianへの書き込み前に、必ず記録内容のプレビューを提示し、ユーザーの承認を得ること。**
- 記録内容はこれまでの会話から自動抽出し、ユーザーが「盛りすぎ」と感じない適切な分量で要約すること。
- Vaultアクセスは `obsidian-cli` (vault="knowledge_base") を使用。直接のファイル操作は禁止。

---

## 🛠 運用フロー（共通プロセス）

1. ユーザーが `/obsidian/` で始まるコマンドを入力する。
2. Claudeは「直近の会話内容」に基づき、指定のフォーマットでプレビュー（下書き）を作成して提示する。
3. Claudeは「この内容でObsidianに記録してよろしいですか？」と確認する。
4. **ユーザーの明確な承諾（はい、yes等）を得てから**、実際にコマンドを実行する。

---

## 📝 Obsidian記録・参照ルール

1. **自動リンク化**: 技術用語（Next.js, AWS, S3等）を自動で `[[ ]]` で囲む。
2. **自動タグ付け**: 内容から `#frontend`, `#aws`, `#learning` 等を推測して付与。
3. **昨日の確認**: ユーザーから状況を問われた際は、`daily:read` で前日のノートを取得し、簡潔に要約・報告すること。

---

## ⚡️ カスタムコマンド（Obsidian名前空間）

### `/obsidian/log` （作業・進捗記録）

- **用途**: 一日の終わりや作業の区切りに、進捗を記録する。
- **コマンド実行時の注意**: `content=` に渡す際、先頭に `\n` を付けること（前エントリーとの空行確保）。テンプレート内の空行は `\n> \n` として渡す。
- **フォーマット**:
  ```
  > [!note]+ HH:MM — [[作業概要]]（HH:MMは現在時刻・JST）
  >
  > **Done:**
  > - [実施内容]
  >
  > **Next:**
  > - [次のアクション]
  >
  > [推測したハッシュタグ]
  ```

### `/obsidian/bug` （エラー・Q&A記録）

- **用途**: エラーの解決や、質問して解決したことを専用ノートとして保存する。
- **保存先ルール**:
  1. 会話から技術スタックを特定（例: `Next.js`, `AWS`, `TypeScript`）
  2. パスを生成: `03_Learning/[技術名]/YYYY-MM-DD_[概要].md`（YYYY-MM-DDは今日の日付）
  3. 同名ファイルが存在する場合は末尾に `_2`, `_3` と連番を付けて衝突を回避
  4. `obsidian create` コマンドで新規ファイルとして保存
- **コマンド**:
  ```
  obsidian create path="03_Learning/[技術名]/YYYY-MM-DD_[概要].md" content="..." vault=knowledge_base
  ```
- **フォーマット**:
  ````
  > [!bug]+ YYYY-MM-DD — [[概要]]
  >
  > **Context:** [何をしていた時に発生したか]
  >
  > > [!failure] Issue
  > > [エラー内容・課題・質問]
  >
  > > [!success] Solution
  > > [解決策・回答]
  >
  > > [!example] Code
  > > ```
  > > [修正コードやコマンド]
  > > ```
  >
  > **Learned:** [次に活かせること]
  >
  > #bug [推測したハッシュタグ]
  ````
